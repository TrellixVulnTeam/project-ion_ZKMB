23 /* how long to boost priority of the main thread for each main
24  loop. This needs to be long enough for all interrupt-level drivers
25  (mostly SPI drivers) to run, and for the main loop of the vehicle
26  code to start the AHRS update.
27 
28  Priority boosting of the main thread in delay_microseconds_boost()
29  avoids the problem that drivers in hpwork all happen to run right
30  at the start of the period where the main vehicle loop is calling
31  wait_for_sample(). That causes main loop timing jitter, which
32  reduces performance. Using the priority boost the main loop
33  temporarily runs at a priority higher than hpwork and the timer
34  thread, which results in much more consistent loop timing.
35 */
